# S3 conditional-write compatibility

Whether each store **enforces** `If-None-Match: *` / `If-Match: <etag>`
on PUT — the primitive this library's commit protocol rests on. A store
that accepts-but-ignores these headers corrupts a lake with zero errors.

Rows marked `ci` are re-verified automatically (weekly + on every
version bump via Renovate); a behavior change fails CI on exactly the
version that changed it. Generated by `scripts/compat_report.py` —
do not edit by hand.
